---
title: "Crear Factura"
sidebarTitle: "Crear Factura"
description: "Crea y emite una factura electronica validada por el SIAT."
api: "POST https://sandbox.cucu.bo/api/v1/invoices"
---

Crea una nueva factura electronica, la firma digitalmente, la envia al SIAT para validacion, genera PDF (A4 y ticket), XML, y opcionalmente envia por email al cliente.

## Headers

<ParamField header="X-API-Key" type="string" required>
  Tu API Key de sandbox (`sk_test_`) o produccion (`sk_live_`).
</ParamField>

<ParamField header="Content-Type" type="string" required>
  `application/json`
</ParamField>

## Request Body

<ParamField body="invoiceType" type="string" required>
  Tipo de factura que determina el documento sector y modalidad. Default: `ELECTRONIC_SALE`.
  Ver [Documentos Sector](/api-reference/overview#tipos-de-documento-sector) para todos los valores.
</ParamField>

<ParamField body="pointOfSaleId" type="string" required>
  UUID del punto de venta. Ejemplo: `660e8400-e29b-41d4-a716-446655440004`
</ParamField>

<ParamField body="clientDocumentType" type="integer" required>
  Tipo de documento del cliente. Valores: `1` (CI), `2` (CEX), `3` (PAS), `4` (OD), `5` (NIT).
</ParamField>

<ParamField body="clientDocumentNumber" type="string" required>
  Numero de documento del cliente (max 50 caracteres).
</ParamField>

<ParamField body="clientBusinessName" type="string" required>
  Razon social del cliente (max 255 caracteres).
</ParamField>

<ParamField body="clientEmail" type="string">
  Email del cliente. Si se proporciona, se envia automaticamente la factura por correo.
</ParamField>

<ParamField body="clientComplement" type="string">
  Complemento del documento de identidad.
</ParamField>

<ParamField body="paymentMethodCode" type="integer" required>
  Metodo de pago. `1` = Efectivo, `2` = Tarjeta, `3` = Cheque, etc.
</ParamField>

<ParamField body="cardNumber" type="string">
  Ultimos 4 digitos de tarjeta (solo si `paymentMethodCode` = 2).
</ParamField>

<ParamField body="exchangeRate" type="number">
  Tipo de cambio (solo si la factura es en moneda extranjera).
</ParamField>

<ParamField body="observations" type="string">
  Observaciones o notas (max 500 caracteres).
</ParamField>

<ParamField body="isTicket" type="boolean" default="false">
  Si `true`, adjunta tambien el formato ticket (80mm) en el email.
</ParamField>

<ParamField body="details" type="array" required>
  Lista de items de la factura (minimo 1).
  <Expandable title="Propiedades de cada item">
    <ParamField body="activityEconomic" type="string" required>
      Codigo de actividad economica (CAEB).
    </ParamField>
    <ParamField body="codeProductSin" type="string" required>
      Codigo de producto/servicio del SIN.
    </ParamField>
    <ParamField body="codeProduct" type="string">
      Codigo interno de tu producto.
    </ParamField>
    <ParamField body="description" type="string" required>
      Descripcion del producto/servicio (max 500 caracteres).
    </ParamField>
    <ParamField body="quantity" type="number" required>
      Cantidad (mayor a 0.00001).
    </ParamField>
    <ParamField body="unitMeasure" type="integer" required>
      Unidad de medida. `58` = Servicio, `1` = Unidad, `57` = Pieza, etc.
    </ParamField>
    <ParamField body="priceUnit" type="number" required>
      Precio unitario (mayor a 0.00001).
    </ParamField>
    <ParamField body="amountDiscount" type="number" default="0">
      Monto de descuento por item.
    </ParamField>
    <ParamField body="serialNumber" type="string">
      Numero de serie del producto.
    </ParamField>
    <ParamField body="imeiNumber" type="string">
      Numero IMEI (solo para telecomunicaciones).
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  `true` si la factura fue emitida exitosamente.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Propiedades">
    <ResponseField name="id" type="string">UUID de la factura.</ResponseField>
    <ResponseField name="invoiceNumber" type="integer">Numero correlativo de factura.</ResponseField>
    <ResponseField name="cuf" type="string">Codigo Unico de Factura (CUF). Identificador ante el SIN.</ResponseField>
    <ResponseField name="state" type="string">Estado SIAT: `VALIDATED`, `REJECTED`, `PENDING`.</ResponseField>
    <ResponseField name="emissionType" type="string">`NORMAL` o `CONTINGENCY`.</ResponseField>
    <ResponseField name="clientBusinessName" type="string">Razon social del cliente.</ResponseField>
    <ResponseField name="amountTotal" type="number">Monto total de la factura.</ResponseField>
    <ResponseField name="literal" type="string">Monto en literal (requerido por SIAT).</ResponseField>
    <ResponseField name="emissionDate" type="string">Fecha y hora de emision.</ResponseField>
    <ResponseField name="pdfUrl" type="string">URL publica del PDF.</ResponseField>
    <ResponseField name="xmlUrl" type="string">URL publica del XML.</ResponseField>
  </Expandable>
</ResponseField>
