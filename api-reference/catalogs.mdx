Los catalogos contienen los codigos oficiales del SIAT que se usan en las facturas (actividades economicas, productos, unidades de medida, etc.).

## Sincronizar todos los catalogos

```
POST /api/v1/siat/sync/all
```

Sincroniza todos los catalogos del SIAT con la base de datos local. Ejecutar al menos una vez al inicio.

<ParamField query="posId" type="string">UUID del punto de venta (opcional, usa el default si se omite).</ParamField>

<CodeGroup>
```bash cURL
curl -X POST https://sandbox.cucu.bo/api/v1/siat/sync/all \
  -H "X-API-Key: YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://sandbox.cucu.bo/api/v1/siat/sync/all', {
  method: 'POST',
  headers: { 'X-API-Key': 'YOUR_API_KEY' }
});
```
</CodeGroup>

---

## Sincronizar catalogo especifico

```
POST /api/v1/siat/sync/{catalog}
```

Catalogos disponibles: `activities`, `products`, `legends`, `cancellation-reasons`, `document-types`, `payment-methods`, `currencies`, `unit-measures`, `significant-events`.

<ParamField path="catalog" type="string" required>Nombre del catalogo a sincronizar.</ParamField>
<ParamField query="posId" type="string">UUID del punto de venta.</ParamField>

---

## Consultar actividades economicas

```
GET /api/v1/siat/sync/catalogs/activities
```

<CodeGroup>
```bash cURL
curl -X GET https://sandbox.cucu.bo/api/v1/siat/sync/catalogs/activities \
  -H "X-API-Key: YOUR_API_KEY"
```
</CodeGroup>

---

## Consultar productos por actividad

```
GET /api/v1/siat/sync/catalogs/products?activityCode=620100
```

<ParamField query="activityCode" type="string">Filtrar productos por codigo de actividad economica.</ParamField>

---

## Consultar leyendas

```
GET /api/v1/siat/sync/catalogs/legends
```

<ParamField query="activityCode" type="string">Filtrar leyendas por actividad economica.</ParamField>

---

## Consultar catalogo generico

```
GET /api/v1/siat/sync/catalogs/{type}
```

Tipos disponibles: `document-types`, `payment-methods`, `currencies`, `unit-measures`, `cancellation-reasons`, `significant-events`.

<ParamField path="type" type="string" required>Tipo de catalogo.</ParamField>

```json
{
  "success": true,
  "data": [
    { "codigo": 1, "descripcion": "Efectivo" },
    { "codigo": 2, "descripcion": "Tarjeta" },
    { "codigo": 3, "descripcion": "Cheque" }
  ]
}
```
